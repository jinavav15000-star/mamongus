# 마몽어스 그림 작업

## 0. 먼저 눈으로 봐라 (건너뛰지 말 것)

코드만 읽고 고치면 반드시 어긋난다. **고치기 전과 후에 각각 뽑아서 비교한다.**

```bash
node build.mjs                                  # src/ → docs/index.html (안 하면 아무 변화 없음)
node tools/shot-char.mjs                        # 양 6색 × 서있기/걷기/시체
node tools/shot-map.mjs test-shots/map.png      # 맵 전체 (서버 불필요, 제일 빠름)
npm run serve & node tools/shot-game.mjs        # 실제 게임 화면
node tools/shot-mg.mjs                          # 미니게임 18종
node tools/shot-ui.mjs                          # 홈·로비·게임·지도
```

**작게 봐서 통해야 진짜다.** 게임 속 양은 30px 안팎이다. 확대해서 예뻐도
`shot-game.mjs` 에서 뭉개지면 실패다. 둘 다 확인할 것.

## 1. 절대 규칙

- **파일은 하나다.** `docs/index.html` 이 곧 게임이고, 용량이 곧 카톡에서의 로딩 시간이다.
  → 이미지 파일·폰트·외부 CDN 금지. **전부 캔버스 도형과 CSS로 그린다.**
- **좌표는 타일 단위**(`TILE=32`). `drawProps` 의 헬퍼들이 픽셀로 바꿔 준다.
- **소품은 충돌 판정이 없다.** 방 경계(타일 좌표)를 넘기면 벽 밖에 떠 보인다.
- **맵 구조(방 위치·크기·복도·벤트·임무 좌표)는 건드리지 않는다.** 검증된 값이다.
  이름·재질·소품·색만 바꾼다.
- 텍스처는 `Render.seeded()` 시드 난수로. `Math.random()` 을 쓰면 프레임마다 흔들린다.

## 2. 캐릭터 (`Render.charShape`, `Render.drawBody`)

원점은 몸통 중심. **구스구스덕처럼 세로로 서서 정면을 본다** — 진행 방향은 호출부의 좌우 반전 + 눈동자 쏠림(lookX)으로만 표현한다. 옆모습으로 되돌리지 말 것 (사용자가 정면형을 골랐다).

지켜야 할 네 가지 (전부 실제로 문제였던 것들):

1. **외곽선은 모든 색에 같은 어두운 색** `#241a12`.
   털색으로 외곽선을 만들면 노랑·하양 양이 짚바닥(`#8a6b3b`)에 묻힌다.
   혹마다 `stroke()` 하면 안쪽에 선이 남는다 →
   **같은 실루엣을 `grow` 만큼 부풀려 어둡게 깔고 그 위에 본체를 얹는다.**
   ```js
   const path = (grow) => { g.beginPath(); g.ellipse(...); for (const [bx,by,br] of BUMPS) { g.moveTo(bx+br+grow, by); g.arc(bx,by,br+grow,0,6.283); } };
   g.fillStyle = OUT; path(2.4); g.fill();
   g.fillStyle = col.hex; path(0); g.fill();
   ```
2. **얼굴 둘레에 밝은 테**(`rgba(255,238,208,.5)`)를 두른다.
   없으면 검정·남색 양은 머리와 몸이 한 덩어리로 뭉개진다.
3. **다리는 양털 아래로 확실히 내린다**(끝 y≈24). 짧으면 떠 있는 털뭉치로 보인다.
   4개를 그리되 뒤쪽 2개는 어둡게 = 깊이감.
4. **음영은 2단계.** 위 밝고 아래 어두운 그라데이션. 하이라이트 하나면 납작하다.

**살아있는 양을 고치면 시체(`drawBody`)도 같이 고친다.** 한쪽만 바꾸면 다른 게임 그림처럼 보인다.

## 3. 맵 (`Render.drawFloors`, `drawProps`)

- 방마다 `floor` 재질이 다르다(`ROOMS` 정의). 재질로 방을 구분하게 하는 게 목적이다.
  이름을 읽어야 어딘지 안다면 실패다.
- 방 이름은 **방 위쪽 나무 간판**으로. 가운데 두면 소품에 가린다.
- 소품 헬퍼: `hay·bale·sack·crate·barrel·trough·tractor·bucket·lantern·bush·post·bench·barrow·tires·stain`
  새 방을 채울 때 먼저 이것들로 조합해 보고, 없으면 추가한다.
- **방이 비어 보이면 잔소품(등불·물통·궤짝·얼룩)을 흩는다.** 큰 소품 하나보다 효과가 크다.

## 4. 색

따뜻한 나무·흙·짚 톤. CSS 변수는 `src/00-style.css` 맨 위.

**절대 바꾸지 않는 색** — 뜻이 있는 색이다:
- 진영: `--goose`(하늘) `--duck`(빨강) `--neut`(보라) — 플레이어가 학습한 구분
- 상태: `--good`(초록) `--bad`(빨강) `--warn`(노랑)
- 온도 조정 미니게임의 파랑·빨강 (차갑다·뜨겁다)

## 5. 연출 (`Render.fx`)

`dustAt` `sparkleAt` `puffAt` `ringAt` + `Render.shake`.
**월드 좌표계에 그리고 안개보다 먼저 그린다** — 벽 너머 연출이 저절로 가려져,
연출을 보고 벽 뒤 사람을 알아채는 새 치트가 생기지 않는다.
**`FX_MAX=180` 상한을 지운다면 그 이유를 설명할 것.** 16명이 동시에 뛰면 저사양 폰이 끊긴다.

## 6. 화면 크기

기준은 **갤럭시 가로 740×360**. 세로가 극단적으로 짧다.
새 UI를 넣으면 이 크기에서 잘리는지 반드시 확인한다. 실제로 여기서만 드러난 문제들:
- 로비 '게임 시작' 버튼이 화면 밖(791px)에 있었다
- 지도 모달이 아래쪽 방들을 잘라먹었다
- 쿨다운 숫자가 버튼 글자를 덮었다

CSS 미디어쿼리는 **`00-style.css` 맨 끝**에 둔다(특이도를 올려주지 않으므로).
