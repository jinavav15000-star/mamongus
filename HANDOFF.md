### 연출 시스템 (같은 날)

`src/60-render.js` 안에 파티클 시스템을 넣었습니다. **월드 좌표계**에 그리고 **안개보다 먼저**
그리므로, 벽 너머에서 일어난 연출은 저절로 가려집니다(치트 방지와 같은 원리).

| 함수 | 쓰임 |
|---|---|
| `Render.dustAt(x, y, dir)` | 걸음 먼지. 진행 방향 **뒤로** 남는다 |
| `Render.sparkleAt(x, y, col)` | 임무 완료 반짝임 (링 1 + 별 10) |
| `Render.puffAt(x, y)` | 벤트 드나들 때 먼지 |
| `Render.ringAt(x, y, col, r)` | 충격파 (킬·사보타주) |
| `Render.shake = n` | 화면 흔들림 (기존 기능) |

- 걸음 먼지는 `draw()` 안에서 **보이는 사람 전원**에게 150ms 간격으로 자동 생성된다
  (`Render.stepT` 가 플레이어별 다음 발자국 시각을 들고 있다)
- **`FX_MAX = 180` 상한이 있다.** 16명이 동시에 뛰면 파티클이 폭증해 저사양 폰이 끊긴다.
  넘치면 오래된 것부터 버린다. 이 상한을 없애지 말 것 (테스트가 지키고 있다)
- 잠긴 문은 빗장이 흐르고(`lineDashOffset`) 숨 쉬듯 깜빡인다
- 화면 전환은 CSS `@keyframes scrIn` (0.22s). 회의·추방·결과는 `scrInBig` (0.34s).
  **`prefers-reduced-motion` 이면 애니메이션을 끈다** — 멀미를 느끼는 사람이 있다
- 연출 관련 테스트 6개가 `test/client-sim.mjs` 의 `▸ 연출` 절에 있다

**연출을 추가할 때**: 소리(`Sfx.*`)와 화면 흔들림(`Render.shake`)은 이미 대부분의 이벤트에
붙어 있다. `src/80-main.js` 의 `onEvent()` 스위치에서 같이 붙이면 된다.

**일부러 안 바꾼 색**
- `--goose:#5fd0ff` / `--duck:#ff5f6d` / `--neut:#c99bff` — **진영 색**이다. 플레이어가 이미 학습한
  구분이라 팔레트 통일보다 식별성이 우선. 갈색 배경 위에서도 세 색이 또렷하게 갈린다
- `--good`(초록) `--bad`(빨강) `--warn`(노랑) — 성공/실패/경고의 관습색
- 온도 조정 미니게임의 파랑/빨강 — 차갑다/뜨겁다는 뜻이라 그대로

## 3. 아키텍처 — 반드시 알아야 할 것

### 네트워크
```
[클라] ──┐
[클라] ──┼──▶ [방장]  게임 로직 권한 · 스냅샷 12Hz 브로드캐스트
[클라] ──┘
```
- **스타 토폴로지** (연결 N−1). 16명 메시는 모바일에서 감당 불가
- **위치는 각자 권한**(입력 지연 0), **판정은 방장 권한**(킬 사거리·임무·투표·승패)
- PeerJS 공개 시그널링 서버 사용, 무료 TURN(openrelay) 폴백

### 🔥 과거에 실제로 터졌던 함정 (같은 실수 반복 금지)

1. **메시지 타입명 충돌 → 무한 재귀**
   `Net.broadcast` 는 방장 자신에게도 emit 된다. 클라→호스트와 호스트→클라가
   같은 타입명을 쓰면 무한 루프. 그래서 채팅은 `chat`(→호스트) / `msg`(→클라) 로 분리.
   지금은 서버발 메시지에 `_s:1` 태그를 달아 구조적으로 차단함.

2. **방장은 `G` 를 로직과 뷰가 공유한다**
   방장이 자기에게 브로드캐스트한 "마스킹된 투표"가 원본 투표를 덮어써 개표가 깨졌었다.
   → 회의 권한 상태는 `Host.M`, 뷰는 `G.meeting` 으로 분리. **다시 합치지 말 것.**
   같은 이유로 `prevPhase` 로 화면 전환을 판단하면 안 된다 (`UI.screen` 으로 판단).

3. **모든 타이머는 방장 시계 기준**
   기기마다 시계가 다르다. `Date.now()` 직접 쓰지 말고 **반드시 `now()`** 사용
   (= `Date.now() + Net.clockOffset`).

4. **CSS 미디어쿼리는 컴포넌트 스타일보다 뒤에 와야 한다**
   미디어쿼리는 특이도를 올려주지 않는다. 가로모드 보정 블록이 앞에 있어서
   회의 화면에 안 먹었던 적 있음. → `00-style.css` **맨 끝**에 위치.

5. **안개(fog)를 메인 캔버스에 `destination-out` 하면 맵까지 지워진다**
   반드시 별도 레이어에서 뚫고 얹을 것. (`Render.fogCv`)

6. **`requestFullscreen` 이 응답 없이 멈출 수 있다**
   `await` 로 기다리면 뒤따르는 방향고정이 영영 실행 안 됨 → 타임아웃으로 분리됨.

7. **전체화면은 입력창 탭에서 발동하면 안 된다**
   키보드가 올라오며 안드로이드가 전체화면을 해제한다. 입력창 탭은 건너뛰고
   전체화면이 풀리면 자동 재무장한다.

## 5. 작업 방식 — 사용자가 기대하는 것

- **한국어로 대화.** 사용자는 개발자가 아님. 전문용어는 풀어서 설명
- **추측하지 말고 실제로 확인할 것.** 브라우저로 띄워 스크린샷을 찍고, 수치를 재고,
  테스트를 돌린 뒤 보고. "고쳤습니다"라고만 하면 신뢰를 잃음
- **버그를 숨기지 말 것.** 발견하면 원인까지 설명. 지금까지 이런 방식으로 신뢰가 쌓임
- **못 하는 건 못 한다고 말할 것.** (예: 아이폰 사파리 전체화면은 애플이 막아서 불가능)
- 큰 변경(공개 저장소 생성 등)은 **먼저 확인**받기
- 사용자는 4인 필요한 게임을 혼자 테스트 못 해 답답해했음 → **봇 기능**을 넣어 해결함.
  앞으로도 "사용자가 직접 확인할 수 있는 수단"을 챙길 것

## 6.4 건초더미 숨기 (2026-07-23 추가, 구스구스덕의 덤불 번안)

- `HIDE_SPOTS` (10-map.js) 7곳 — **drawProps 의 실제 건초 소품 좌표와 일치해야 한다**
- 벤트와 같은 계열: 숨으면 서버가 스냅샷에서 제외 → 콘솔로도 못 찾는다
- **한 버튼(🌾 숨기)으로 두 동작**: 빈 더미 = 숨기 / 누가 있는 더미 = 튀어나오게(수색)
  → 늑대가 수상한 더미를 뒤지는 심리전. 빈 더미 수색 = 시간 낭비가 카운터
- 숨는 동안: 이동·킬·임무·신고 전부 차단(호스트 강제), 시야 0.55배, 위치 더미에 고정
- 이동 입력 시 자동으로 나옴. 회의·라운드 재개·사망 시 강제 해제 (ventId 초기화 지점과 동일)
- 연출: rustle(부스럭, 근처만) / flushed(지푸라기 폭발) — 누가 숨었는지는 방송하지 않는다

## 6.5 대기실·채팅 구조 (2026-07-23 추가)

- **대기실 = 맵이다** (구스구스덕 방식). 로비 페이즈에도 이동·스냅샷이 돈다
  - 호스트: `onPos`·`sendSnap` 이 lobby 페이즈 허용. 로비 스냅샷은 시야 컬링 없이 전원→전원
  - 클라: phase lobby → `#screen-game` + `.lobby-mode` 클래스(게임 HUD 숨김) + `#lobby-hud`
  - 옛 로비 화면(`#screen-lobby`)은 ☰ 버튼으로 여는 **오버레이 패널**이 됐다 (z-index 40)
- **게임 중 채팅 허용** (`onChat` 의 산 사람 차단 제거). 죽은 사람은 여전히 유령 채널만
- **말풍선**: `Meeting.addMsg` 가 `G.players[pid].bubble` 을 채우고 `drawDuck` 이 그린다.
  보이는 사람만 그려지므로 말풍선으로 벽 뒤 위치가 새지 않는다
- **캐릭터는 정면형**(구스구스덕 비례). 사용자가 여러 차례 다듬어 고른 모습이니
