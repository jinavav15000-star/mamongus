## 3. 아키텍처 — 반드시 알아야 할 것

### 네트워크
```
[클라] ──┐
[클라] ──┼──▶ [방장]  게임 로직 권한 · 스냅샷 12Hz 브로드캐스트
[클라] ──┘
```
- **스타 토폴로지** (연결 N−1). 16명 메시는 모바일에서 감당 불가
- **위치는 각자 권한**(입력 지연 0), **판정은 방장 권한**(킬 사거리·임무·투표·승패)
- PeerJS 공개 시그널링 서버 사용, 무료 TURN(openrelay) 폴백

### 🔥 과거에 실제로 터졌던 함정 (같은 실수 반복 금지)

1. **메시지 타입명 충돌 → 무한 재귀**
   `Net.broadcast` 는 방장 자신에게도 emit 된다. 클라→호스트와 호스트→클라가
   같은 타입명을 쓰면 무한 루프. 그래서 채팅은 `chat`(→호스트) / `msg`(→클라) 로 분리.
   지금은 서버발 메시지에 `_s:1` 태그를 달아 구조적으로 차단함.

2. **방장은 `G` 를 로직과 뷰가 공유한다**
   방장이 자기에게 브로드캐스트한 "마스킹된 투표"가 원본 투표를 덮어써 개표가 깨졌었다.
   → 회의 권한 상태는 `Host.M`, 뷰는 `G.meeting` 으로 분리. **다시 합치지 말 것.**
   같은 이유로 `prevPhase` 로 화면 전환을 판단하면 안 된다 (`UI.screen` 으로 판단).

3. **모든 타이머는 방장 시계 기준**
   기기마다 시계가 다르다. `Date.now()` 직접 쓰지 말고 **반드시 `now()`** 사용
   (= `Date.now() + Net.clockOffset`).

4. **CSS 미디어쿼리는 컴포넌트 스타일보다 뒤에 와야 한다**
   미디어쿼리는 특이도를 올려주지 않는다. 가로모드 보정 블록이 앞에 있어서
   회의 화면에 안 먹었던 적 있음. → `00-style.css` **맨 끝**에 위치.

5. **안개(fog)를 메인 캔버스에 `destination-out` 하면 맵까지 지워진다**
   반드시 별도 레이어에서 뚫고 얹을 것. (`Render.fogCv`)

6. **`requestFullscreen` 이 응답 없이 멈출 수 있다**
   `await` 로 기다리면 뒤따르는 방향고정이 영영 실행 안 됨 → 타임아웃으로 분리됨.

7. **전체화면은 입력창 탭에서 발동하면 안 된다**
   키보드가 올라오며 안드로이드가 전체화면을 해제한다. 입력창 탭은 건너뛰고
   전체화면이 풀리면 자동 재무장한다.

### 치트 방지
좌표는 **방장이 시야를 계산해 보이는 사람만** 전송. 콘솔을 열어도 시야 밖 위치·범인·
벤트 사용자·추방자 직업을 알 수 없다. **이 원칙을 깨는 코드를 넣지 말 것.**

## 5. 작업 방식 — 사용자가 기대하는 것

- **한국어로 대화.** 사용자는 개발자가 아님. 전문용어는 풀어서 설명
- **추측하지 말고 실제로 확인할 것.** 브라우저로 띄워 스크린샷을 찍고, 수치를 재고,
  테스트를 돌린 뒤 보고. "고쳤습니다"라고만 하면 신뢰를 잃음
- **버그를 숨기지 말 것.** 발견하면 원인까지 설명. 지금까지 이런 방식으로 신뢰가 쌓임

- 사용자는 4인 필요한 게임을 혼자 테스트 못 해 답답해했음 → **봇 기능**을 넣어 해결함.
