# 🐑 마몽어스 (MamongUs)

브라우저에서 바로 하는 **양들의 마피아** 게임. 설치·회원가입 없이 **링크 하나만 공유하면** 최대 16명이 같이 합니다.
Goose Goose Duck / Among Us 스타일 — **22개 직업**, **18종 임무 미니게임**, **사보타주**, **텍스트 회의·투표**.

> **음성 없이 플레이하는 것을 전제로 설계했습니다.** 자동 동선 기록, 퀵챗, 목격 정보 표시로
> 타이핑 없이도 추리와 변론이 가능합니다. (음성은 원하는 사람끼리 켤 수 있습니다.)

## 🚀 카카오톡에 공유할 주소 만들기

### 방법 1 · GitHub Pages — 무료·영구 (권장)

이 저장소는 이미 **`/docs` 폴더 배포**에 맞춰져 있습니다.

저장소 → **Settings → Pages → Source: `Deploy from a branch` → `main` / `/docs`** 로 설정.
1~2분 뒤 아래 주소에서 열립니다.

```
https://<깃허브아이디>.github.io/<저장소이름>/
```

소스를 고친 뒤에는 `npm run build` 로 `docs/index.html` 을 다시 만들고 커밋·푸시하면 즉시 반영됩니다.

### 방법 2 · Netlify Drop — 가장 빠름 (회원가입 없이 1분)

1. <https://app.netlify.com/drop> 접속
2. `docs` 폴더를 브라우저 창에 **드래그 앤 드롭**
3. 바로 `https://무작위이름.netlify.app` 주소가 생깁니다

> ⚠️ 회원가입 없이 올리면 사이트가 일정 시간 뒤 사라집니다. 계속 쓰려면 무료 가입 후 저장하세요.

### 방법 3 · Cloudflare Pages / Vercel
빌드 명령 없음, 출력 디렉터리 `docs` 로 지정하면 됩니다.

### ✅ 공유 전 체크리스트
- [ ] 주소가 **https://** 로 시작하나요? (WebRTC는 HTTPS에서만 동작합니다)
- [ ] 휴대폰에서 열어봤나요?
- [ ] 방장은 **PC 또는 화면이 꺼지지 않는 기기**로 하는 게 안정적입니다

1. 아무나 **방 만들기** → 4자리 코드와 초대 링크가 생성됩니다

## 🗣️ 음성 없이 잘하는 법 (이 게임의 핵심 설계)

대부분 음성을 못 쓰는 상황을 전제로 만든 기능들입니다.

| 기능 | 설명 |
|---|---|
| **📍 내 동선** | 지나온 방과 시각이 자동 기록됩니다. 회의에서 버튼 한 번이면 `00:12 카페테리아 → 00:34 전기실 → …` 이 채팅에 그대로 들어갑니다. 알리바이를 타이핑할 필요가 없습니다. |
| **퀵챗** | `[색] 의심돼`, `나 [방]에 있었어` 같은 문장을 누르면 색·방 칩이 떠서 **두 번 탭으로 문장 완성**. 6개 카테고리 50여 문장. |
| **👁 목격 정보** | 투표 화면의 각 플레이어 카드에 **내가 그 사람을 마지막으로 본 시각과 장소**가 자동 표시됩니다. |
| **자동 시스템 로그** | 시체 발견·사망자·발견 장소가 채팅에 자동으로 남습니다. |
| **넉넉한 시간** | 토론 60초 / 투표 75초가 기본값입니다 (Among Us 대비 길게). 설정에서 조절 가능. |
| **투표 비공개** | 개표 전에는 "누가 투표했는지"만 보이고 "누구에게"는 가려집니다. 눈치싸움 방지. |
| **🧭 길안내** | 화면 가장자리 화살표가 다음 임무·수리 지점 방향과 거리를 알려줍니다. 처음 하는 사람이 방을 못 찾는 문제를 해결합니다. (메뉴에서 끌 수 있음) |
| **👻 유령 채팅** | 죽어도 유령끼리 대화할 수 있고, **영매**가 그 대화를 듣습니다. |

**음성을 쓰고 싶다면**: `☰ 메뉴 → 🎙️ 음성채팅 켜기`.
켠 사람끼리만 연결되고, **거리에 따라 소리가 줄어드는 근접 음성**입니다. 회의 중에는 전원에게 들립니다.
유령의 목소리는 산 사람에게 들리지 않습니다.

### 네트워크 구조

```
[클라] ──┐
[클라] ──┼──▶ [호스트]  게임 로직 권한 · 스냅샷 12Hz 브로드캐스트
[클라] ──┘
```

- **스타 토폴로지**: 연결 수 N−1. 16명 메시(120연결)는 모바일에서 감당 못 합니다.
- **위치는 각자 권한**(입력 지연 0), **판정은 호스트 권한**(킬 사거리·임무·투표·승패).
- **시계 동기화**: 기기마다 시계가 다르므로 ping RTT로 오프셋을 추정해 모든 타이머를 호스트 시각에 맞춥니다.
- **재접속**: `sessionStorage` 기반 uid로 새로고침·연결 끊김 시 원래 슬롯으로 복귀합니다.
- **NAT 통과**: STUN 3개 + 무료 TURN 릴레이(openrelay) 폴백.

### 방장 교체 (호스트 마이그레이션)

방장이 브라우저를 닫아도 게임이 끝나지 않습니다.

- 방장은 **후계자 1명**에게만 권한 상태(역할·임무·시체·사보타주·회의)를 2.5초마다 넘겨둡니다.
- 방장이 사라지면 후계자가 `방코드-g1` 로 방을 다시 열고, 나머지는 자동으로 재접속합니다.
- 시각 기준이 바뀌므로 모든 타이머를 새 방장 시계로 환산합니다.
- 전원에게 뿌리지 않는 이유: 모든 역할이 노출되기 때문입니다. 신뢰 범위는 방장+후계자 2명입니다.

### 치트 방지

- 좌표는 **서버(방장)가 시야를 계산해 보이는 사람만** 보냅니다. 콘솔을 열어도 시야 밖 위치는 없습니다.
- 관리실·감시카메라 정보도 방장이 계산한 결과만 전송합니다.
- 범인·벤트 사용자·추방자 직업은 해당 대상에게만 개별 전송합니다.

### 알려진 제약

- 방장과 후계자가 **동시에** 끊기면 게임이 종료됩니다 (이중 장애).
- 방장이 다른 앱으로 넘어가 있는 동안에는 모두의 화면이 느려집니다 (브라우저가 백그라운드 탭을 제한).
- 대칭형 NAT 환경에서는 TURN 릴레이를 타므로 지연이 늘 수 있습니다.
- 카카오톡 인앱 브라우저에서 마이크 권한이 막히는 경우가 있습니다 → 음성은 외부 브라우저 권장.
